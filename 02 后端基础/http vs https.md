HTTP 与 HTTPS 超清晰对比笔记（面试必考）

## 一、核心定义对比

**HTTP**：超文本传输协议，**明文、无加密、无安全校验**，基于 TCP，用于网页与接口数据传输。

**HTTPS**：安全超文本传输协议 = **HTTP + TLS/SSL 加密层**，密文传输、可认证、防篡改，安全可靠。

## 二、全方位详细对比表（重点背诵）

| 对比维度       | HTTP           | HTTPS             |
| ---------- | -------------- | ----------------- |
| **默认端口**   | 80             | 443               |
| **传输方式**   | 明文传输，数据裸奔      | TLS/SSL 加密传输（密文）  |
| **安全性**    | 极低，可窃听、篡改、伪装   | 高，防窃听、防篡改、防伪造     |
| **加密机制**   | 无加密            | 非对称加密握手 + 对称加密传数据 |
| **身份认证**   | 无认证，无法校验服务端真伪  | CA 证书校验服务器身份，防钓鱼  |
| **数据完整性**  | 无校验，数据可被中间人篡改  | 哈希校验，数据一旦篡改立即失效   |
| **证书要求**   | 无需证书           | 必须 CA 数字证书        |
| **性能开销**   | 小，直接传输，速度快     | 略大，需要握手、加解密运算     |
| **搜索引擎权重** | 低，现代浏览器标记“不安全” | 高，搜索引擎优先收录        |
| **适用场景**   | 公开静态资源、内网测试    | 网站、登录、支付、接口、生产环境  |

## 三、HTTPS 解决了 HTTP 的哪三个问题？（面试高频）

- **解决窃听**：数据加密，中间人抓包无法读取明文

- **解决篡改**：消息摘要校验，数据一旦改动无法通过校验

- **解决伪装**：CA 证书权威认证，杜绝伪造服务器、钓鱼站点

## 四、HTTPS 加密流程极简总结

1. 客户端与服务端协商加密套件、TLS 版本

2. 服务端下发 CA 证书，客户端验证合法性

3. 客户端用服务端公钥加密预主密钥并发送

4. 双方协商生成统一的**会话密钥（对称密钥）**

5. 后续所有 HTTP 数据全部通过会话密钥加密传输

## 五、一句话终极区别（面试秒杀句）

**HTTP 是明文裸奔传输，不安全；HTTPS 在 HTTP 基础上增加 TLS/SSL 加密层，通过非对称加密协商密钥、对称加密传输数据，实现加密、防篡改、身份认证，保障通信安全。**

## 六、常见面试问答精简

**Q：HTTPS 为什么比 HTTP 安全？**

因为 HTTPS 并非新协议，只是 HTTP 的加密版本，通过 TLS 握手实现数据加密传输、完整性校验、服务器身份认证，解决了 HTTP 明文传输的中间人攻击问题。

**Q：HTTPS 为什么速度比 HTTP 慢？**

多了 TLS 四次握手、证书校验、加解密运算，存在少量 CPU 和网络开销。

**Q：HTTPS 一定绝对安全吗？**

不是。证书过期、证书伪造、私钥泄露、旧加密算法漏洞依然可能导致安全风险。
