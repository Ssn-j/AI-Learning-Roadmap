# 异常处理

程序运行时发生的错误叫异常。异常处理让程序遇到错误时不会崩溃，而是能够优雅地处理。

## 1. 什么是异常

**定义**：程序运行时发生的错误，如果不处理，程序会崩溃并停止执行。

### 常见异常类型

| 异常类型                | 触发场景    | 示例                |
| ------------------- | ------- | ----------------- |
| `ValueError`        | 类型转换失败  | `int("abc")`      |
| `TypeError`         | 操作类型错误  | `"1" + 1`         |
| `KeyError`          | 字典键不存在  | `d["不存在的键"]`      |
| `IndexError`        | 列表索引越界  | `lst[100]`        |
| `AttributeError`    | 对象没有该属性 | `None.name`       |
| `FileNotFoundError` | 文件不存在   | `open("不存在.txt")` |
| `ZeroDivisionError` | 除以零     | `1 / 0`           |
| `ImportError`       | 导入模块失败  | `import 不存在的模块`   |
| `ConnectionError`   | 网络连接失败  | 请求不存在的URL         |
| `TimeoutError`      | 操作超时    | 网络请求超时            |

## 2. 基本语法



| 语法结构          | 作用          |
| ------------- | ----------- |
| `try`         | 包裹可能报错的代码   |
| `except 异常类型` | 捕获指定异常并处理   |
| `except`      | 捕获所有异常（通用）  |
| `else`        | 无异常时执行      |
| `finally`     | 无论是否异常，都会执行 |

### 2.1 `try` / `except`

异常捕捉可以使用 try/except 语句。

```python
try:
    # 代码
except 异常类型:
    # 发生异常时执行的代码
```

示例：

```python
try:
    # 可能会出错的代码
    num = int(input("请输入数字: "))
    result = 100 / num
    print(f"结果是: {result}")
except ValueError:
    # 发生 ValueError 时执行
    print("请输入有效的数字")
except ZeroDivisionError:
    # 发生 ZeroDivisionError 时执行
    print("除数不能为0")
```

### 2.2 `else` 和 `finally`

| 关键字       | 执行时机                    |
| --------- | ----------------------- |
| `else`    | 没有发生任何异常时执行             |
| `finally` | 无论是否发生异常，都会执行（通常用于清理资源） |

python

```python
try:
    file = open("data.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("文件不存在")
else:
    print(f"文件内容长度: {len(content)}")  # 没有异常时执行
finally:
    file.close()  # 无论是否异常，都关闭文件
    print("文件已关闭")
```

## 4. 捕获所有异常

python

```python
try:
    risky_operation()
except Exception as e:  # 捕获所有异常
    print(f"发生错误: {e}")
```

**注意**：尽量不要直接用 `except:`（不加 `Exception`），因为会捕获 `KeyboardInterrupt`、`SystemExit` 等系统异常。

```python
# 不推荐：会捕获系统异常

try:
 while True:
 pass
except:
 print("被捕获") # Ctrl+C 也会被捕获，程序无法终止

# 推荐：明确捕获 Exception

try:
 risky_operation()
except Exception as e:
 print(f"错误: {e}")
```



## 5. `raise`：主动抛出异常

python

def validate_age(age):
    if age < 0:
        raise ValueError("年龄不能为负数")
    if age > 150:
        raise ValueError("年龄超出合理范围")
    return age
try:
    validate_age(-5)
except ValueError as e:
    print(f"验证失败: {e}")

## 6. 自定义异常

python

class ValidationError(Exception):
    """自定义验证异常"""
    pass
class NetworkError(Exception):
    """自定义网络异常"""
    pass
def fetch_data():
    raise NetworkError("网络连接超时")
try:
    fetch_data()
except NetworkError as e:
    print(f"网络错误: {e}")

## 7. `assert`：断言

用于调试，条件为 `False` 时抛出 `AssertionError`。

python

def divide(a, b):
    assert b != 0, "除数不能为0"
    return a / b
divide(10, 0)  # AssertionError: 除数不能为0

**注意**：生产环境中可以用 `-O` 参数关闭断言，所以不要依赖断言做数据校验。

## 8. 异常链：`raise from`

保留原始异常信息，便于排查问题。

python

try:
    result = int("abc")
except ValueError as e:
    raise RuntimeError("转换失败") from e

# 输出会显示原始异常和转换后的异常

## 9. 实际应用场景

### 场景1：API 调用异常处理

python

import requests
def call_api(url):
    try:
        response = requests.get(url, timeout=5)
        response.raise_for_status()  # 4xx/5xx 会抛异常
        return response.json()
    except requests.exceptions.Timeout:
        print("请求超时，请稍后重试")
    except requests.exceptions.ConnectionError:
        print("网络连接失败")
    except requests.exceptions.HTTPError as e:
        print(f"HTTP错误: {e}")
    except Exception as e:
        print(f"未知错误: {e}")
    return None

### 场景2：文件操作异常处理

python

def read_file_safe(filepath):
    try:
        with open(filepath, "r", encoding="utf-8") as f:
            return f.read()
    except FileNotFoundError:
        print(f"文件不存在: {filepath}")
    except PermissionError:
        print(f"没有权限读取: {filepath}")
    except UnicodeDecodeError:
        print(f"文件编码错误: {filepath}")
    except Exception as e:
        print(f"读取失败: {e}")
    return None

### 场景3：重试机制

python

import time
def retry(func, max_attempts=3, delay=1):
    for attempt in range(max_attempts):
        try:
            return func()
        except Exception as e:
            print(f"第 {attempt + 1} 次尝试失败: {e}")
            if attempt == max_attempts - 1:
                raise
            time.sleep(delay)

# 使用示例

result = retry(lambda: call_api("https://api.example.com"))
