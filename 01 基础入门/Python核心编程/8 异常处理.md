# 异常处理

程序运行时发生的错误叫异常。异常处理让程序遇到错误时不会崩溃，而是能够优雅地处理。

## 1 什么是异常

**定义**：程序运行时发生的错误，如果不处理，程序会崩溃并停止执行。

### 常见异常类型

| 异常类型                | 触发场景    | 示例                |
| ------------------- | ------- | ----------------- |
| `ValueError`        | 类型转换失败  | `int("abc")`      |
| `TypeError`         | 操作类型错误  | `"1" + 1`         |
| `KeyError`          | 字典键不存在  | `d["不存在的键"]`      |
| `IndexError`        | 列表索引越界  | `lst[100]`        |
| `AttributeError`    | 对象没有该属性 | `None.name`       |
| `FileNotFoundError` | 文件不存在   | `open("不存在.txt")` |
| `ZeroDivisionError` | 除以零     | `1 / 0`           |
| `ImportError`       | 导入模块失败  | `import 不存在的模块`   |
| `ConnectionError`   | 网络连接失败  | 请求不存在的URL         |
| `TimeoutError`      | 操作超时    | 网络请求超时            |

## 2 异常处理

| 语法结构          | 作用          |
| ------------- | ----------- |
| `try`         | 包裹可能报错的代码   |
| `except 异常类型` | 捕获指定异常并处理   |
| `except`      | 捕获所有异常（通用）  |
| `else`        | 无异常时执行      |
| `finally`     | 无论是否异常，都会执行 |

### 2.1 try/ except 异常类型

异常捕捉可以使用 try/except 语句。

语法：

```python
try:
    # 代码
except 异常类型:
    # 发生异常时执行的代码
```

try 语句按照如下方式工作；

- 首先，执行 try 子句（在关键字 try 和关键字 except 之间的语句）。

- 如果没有异常发生，忽略 except 子句，try 子句执行后结束。

- 如果在执行 try 子句的过程中发生了异常，那么 try 子句余下的部分将被忽略。如果异常的类型和 except 之后的名称相符，那么对应的 except 子句将被执行。

- 如果一个异常没有与任何的 except 匹配，那么这个异常将会传递给上层的 try 中。

### 2.2  try/except...else

try/except 语句还有一个可选的 **else** 子句，如果使用这个子句，那么必须放在所有的 except 子句之后。

else 子句将在 try 子句没有发生任何异常的时候执行。

使用 else 子句比把所有的语句都放在 try 子句里面要好，这样可以避免一些意想不到，而 except 又无法捕获的异常。

语法：

```python
try:
    # 代码
except 异常类型:
    # 发生异常时执行的代码
else：
    # 没有异常时执行的代码
```

### 2.3 try-finally 语句

**作用**：无论是否触发异常，**代码必定执行**，用于**收尾、清理操作**。

规则：异常未被捕获时，先执行`finally`，再抛出异常。

语法：

```python
try:
    # 代码
except 异常类型:
    # 发生异常时执行的代码
else：
    # 没有异常时执行的代码
finally:
    # 不管有没有异常都会执行的代码
```

示例：

```python
try:
    # 可能会出错的代码
    num = int(input("请输入数字: "))
    result = 100 / num
    print(f"结果是: {result}")
except ValueError:
    # 发生 ValueError 时执行
    print("请输入有效的数字")
except ZeroDivisionError:
    # 发生 ZeroDivisionError 时执行
    print("除数不能为0")
```

### 2.4 捕获所有异常

| 写法                      | 捕获范围            | 问题 / 特点                    | 推荐程度  |
| ----------------------- | --------------- | -------------------------- | ----- |
| `except Exception as e` | 仅捕获程序异常，不拦截系统异常 | 可正常使用`Ctrl+C`终止程序，安全规范     | ✅ 推荐  |
| `except:`（空捕获）          | 捕获所有异常，包含系统级异常  | `Ctrl+C`无法终止程序，易隐藏错误，不利于调试 | ❌ 不推荐 |

**注意**：尽量不要直接用 `except:`（不加 `Exception`），因为会捕获 `KeyboardInterrupt`、`SystemExit` 等系统异常。

示例：

```python
# 推荐写法
try:
    1/0
except Exception as e:
    print("错误：", e)

# 不推荐写法
try:
    while True:
        pass
except:
    print("触发捕获")
```

## 3  `raise`：主动抛出异常

### 3.1 `raise 异常` = **主动报错**

raise = 主动抛出异常 + 自定义提示

语法：

```python
raise 异常类型(提示信息)
```

注意：

- `raise` 的参数必须是：**异常类** 或 **异常实例**
- 不能随便抛数字、字符串，只能抛**异常**

示例：

```python
age = -3
if age < 0:
    raise ValueError("年龄不能为负数")
```

### 3.2 `raise`（无参数）= **把异常再抛出去**

**作用**：捕获异常后，**不处理，直接再次抛出**（交给上层处理）

```python
try:
    raise NameError("出错啦")
except NameError:
    print("捕获到异常")
    raise  # 再次抛出同一个异常
```

### 3.3 抛异常

- 抛异常：程序**上报运行错误**，中断正常执行。
- 自动抛：运行出错时 Python 默认行为。
- 手动抛：用 `raise` 主动上报逻辑错误。
- 单独 `raise`：接住异常后，**再次向外抛出**。

## 4 自定义异常

自定义异常 = 继承 Exception + 自定义提示 + 可精准捕获处理

### 4.1 定义

自定义异常需要**继承自 `Exception`**。

### 4.2 基础格式

```python
class 自定义异常名(Exception):
    def __init__(self, value):
        self.value = value
    def __str__(self):
        return repr(self.value)
```

### 4.3 使用

```python
try:
    raise 自定义异常名("提示信息")
except 自定义异常名 as e:
    print(e.value)
```

### 4.4 规范用法（项目常用）

- 创建**基础异常类**
- 不同错误继承基础类，形成异常体系
- 异常名建议以 **Error 结尾**

```python
# 基础异常
class Error(Exception):
    pass

# 子异常
class InputError(Error):
    def __init__(self, expression, message):
        self.expression = expression
        self.message = message
```

## 5 定义清理行为

| 关键字       | 核心作用                    | 使用场景            | 特点                |
| --------- | ----------------------- | --------------- | ----------------- |
| `finally` | **强制执行清理代码**，无论是否异常都会运行 | 关闭文件、释放连接、收尾操作  | 配合 `try` 使用，百分百执行 |
| `with`    | **自动资源管理**，自带清理逻辑       | 文件操作、网络请求、数据库连接 | 语法简洁、自动关闭、无需手动处理  |

### 5.1 `finally` 子句

**作用**：无论是否触发异常，**代码必定执行**，用于收尾、清理操作。

规则：异常未被捕获时，先执行`finally`，再抛出异常。

```python
try:
    raise KeyboardInterrupt
finally:
    print("总会执行")
```

### 5.2 `with` 语句（预定义清理）

**作用**：自动完成资源清理（如关闭文件、连接），出错也能正常释放资源。

适用：文件、网络连接等需要手动关闭的对象。

```python
# 自动关闭文件，无需手动 f.close()
with open("myfile.txt") as f:
    for line in f:
        print(line, end="")
```

## 6 `assert`：断言

**assert = 条件不成立就报错，用于开发阶段自检。**

### 6.1 作用

用于判断条件是否成立，条件为 False 时直接抛出异常，提前拦截错误。

### 6.2 语法

```python
assert 表达式
assert 表达式, "自定义错误信息"
```

等加代码：

```python
if not 表达式:
    raise AssertionError(错误信息)
```

### 6.3 用途

- 检查参数合法性
- 检查系统环境
- 开发调试时提前发现错误

示例：

```python
assert 1 == 2, "1不等于2"  # 条件不成立 → 报错
```

## 7 异常链：`raise from`

`raise from` = 新异常 + 保留原始异常，形成完整异常链，方便排查问题。

### 7.1 作用

 把 “原始异常” 和 “新异常” 绑定在一起，保留完整报错链，方便定位根因。

### 7.2 语法

```python
raise 新异常 from 原始异常
```

### 7.3 效果

报错会同时显示：**新异常**（你抛出的）、**原始异常**（真正出错的原因）

方便调试，**不会丢失错误根源**。

示例：

```python
try:
    int("abc")       # 原始错误：ValueError
except ValueError as e:
    # 抛出新异常，并关联原始异常
    raise RuntimeError("格式转换失败") from e
```

## 8 语法错误VS异常

| 分类   | 定义                         | 触发时机           | 典型原因              | 运行表现                        | 示例 & 报错类型                                                                                      |
| ---- | -------------------------- | -------------- | ----------------- | --------------------------- | ---------------------------------------------------------------------------------------------- |
| 语法错误 | 代码书写不符合 Python 语法规则，又称解析错误 | 程序运行前，代码解析阶段   | 漏符号、关键字误用、缩进错误等   | 代码完全无法执行，解释器直接拒绝运行          | `while True print('Hello')`<br>`SyntaxError: invalid syntax`                                   |
| 异常   | 代码语法合法，运行过程中出现逻辑 / 操作错误    | 程序运行中，执行对应代码行时 | 除零运算、变量未定义、类型不匹配等 | 程序可正常启动，执行到出错语句才中断，支持异常捕获处理 | 1. `10*(1/0)` → `ZeroDivisionError`<br>2. `4+spam*3` → `NameError`<br>3. `'2'+2` → `TypeError` |

## 9 应用场景

| 应用场景     | 处理要点                                       | 核心说明                |
| -------- | ------------------------------------------ | ------------------- |
| API 调用异常 | 细分捕获超时、连接、HTTP 状态异常；使用`raise_for_status()` | 区分网络、请求状态类错误，针对性提示  |
| 文件操作异常   | 捕获文件不存在、权限不足、编码错误等                         | 精准处理文件读写常见问题，避免程序崩溃 |
| 重试机制     | 循环重试，超限后重新抛出异常                             | 失败后延时重试，多次失败则向上传递异常 |

### 场景1：API 调用异常处理

```python
import requests
def call_api(url):
 try:
 response = requests.get(url, timeout=5)
 response.raise_for_status() # 4xx/5xx 会抛异常
 return response.json()
 except requests.exceptions.Timeout:
 print("请求超时，请稍后重试")
 except requests.exceptions.ConnectionError:
 print("网络连接失败")
 except requests.exceptions.HTTPError as e:
 print(f"HTTP错误: {e}")
 except Exception as e:
 print(f"未知错误: {e}")
 return None
```

### 场景2：文件操作异常处理

```python
def read_file_safe(filepath):
 try:
 with open(filepath, "r", encoding="utf-8") as f:
 return f.read()
 except FileNotFoundError:
 print(f"文件不存在: {filepath}")
 except PermissionError:
 print(f"没有权限读取: {filepath}")
 except UnicodeDecodeError:
 print(f"文件编码错误: {filepath}")
 except Exception as e:
 print(f"读取失败: {e}")
 return None
```

### 场景3：重试机制

```python
import time
def retry(func, max_attempts=3, delay=1):
 for attempt in range(max_attempts):
 try:
 return func()
 except Exception as e:
 print(f"第 {attempt + 1} 次尝试失败: {e}")
 if attempt == max_attempts - 1:
 raise
 time.sleep(delay)

# 使用示例

result = retry(lambda: call_api("[https://api.example.com"](https://api.example.com")))
```
