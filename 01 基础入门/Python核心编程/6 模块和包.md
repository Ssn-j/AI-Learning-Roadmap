# 模块与包

模块是 Python 组织代码的基本单位。随着项目变大，把代码拆分成多个文件（模块）是必须的。

## 1. 模块（Module）

**定义**：模块（Module）是一个包含 Python 定义和语句的文件，一个 `.py` 文件就是一个模块，模块名就是文件名（不含 `.py`）。

作用：

- **代码复用**：将常用的功能封装到模块中，可以在多个程序中重复使用。

- **命名空间管理**：模块可以避免命名冲突，不同模块中的同名函数或变量不会互相干扰。

- **代码组织**：将代码按功能划分到不同的模块中，使程序结构更清晰。

### 1.1 导入模块

| 导入方式   | 语法                                                | 优点        | 缺点 / 注意        |
| ------ | ------------------------------------------------- | --------- | -------------- |
| 导入整个模块 | `import 模块名`                                      | 清晰、不冲突    | 使用时需加**模块名前缀** |
| 导入指定部分 | `from 模块 import 名称1,名称2`                          | 简洁、直接使用   | 导入过多易混乱        |
| 导入全部内容 | `from 模块 import *`                                | 代码最少      | **不推荐**，易命名冲突  |
| 起别名    | `import 模块 as 别名`<br><br>`from 模块 import 名 as 别名` | 解决重名、简化长名 | 别名需见名知意        |

**注意：** 一个模块只会被导入一次，不管你执行了多少次 `import`。这样可以防止导入模块被一遍又一遍地执行。

#### 1.1.1 import 语句：导入整个模块

语法：

```python
import 模块1，模块2...
```

#### 1.1.2 from … import 语句：导入模块的指定部分

语法：

```python
from 模块 import 名字
```

#### 1.1.3 from … import * 语句

**不推荐，容易引起命名冲突。**

语法：

```python
from 模块 import *
```

#### 1.1.4 给模块起别名

使用 `as`关键字为模块或函数起别名：

```python
import numpy as np  # 将 numpy 模块别名设置为 np
from math import sqrt as square_root  # 将 sqrt 函数别名设置为 square_root
```

#### 1.1.5 示例

```pythonag-0-1jproedmjag-1-1jproedmj
# 方式1：导入整个模块
import math
print(math.sqrt(16))  # 输出：4.0
```

### 1.2 模块搜索顺序

（1）Python 导入模块时，按以下顺序查找：

- 当前目录

- 环境变量 `PYTHONPATH` 指定的目录

- Python 安装目录（标准库）

- 第三方库安装目录（site-packages）、`.pth` 文件中指定的目录。

（2）sys.path：查看所有搜索路径

```python
import sys
print(sys.path) # 查看所有搜索路径
```

### 1.3 `__name__` 内置变量

（1）**作用**：标识当前模块名称

（2）**取值规则**

- 模块**直接运行（作为主程序）**：`__name__ = "__main__"`
- 模块**被其他文件导入**：`__name__ = 模块名`（不含 `.py` 后缀）

（3）**`__main__` 说明**

特殊标识字符串，代表当前文件是**入口主程序**。

```python
# module.py
print(f"模块名：{__name__}")
# 直接运行输出：模块名：__main__
```

（4）经典用法

作用：模块被导入时，不会执行此处测试 / 调用代码。

```python
if __name__ == "__main__":
    # 仅文件作为主程序运行时，内部代码才执行
    main()
```

### 1.4 dir() 函数

内置的函数 dir() 可以找到模块内定义的所有名称。以一个字符串列表的形式返回:

#### 1.4.1 查看模块

语法：

```pyrhon
dir(模块)
```

#### 1.4.2 查看当前文件

语法：

```python
dir()
```

**示例：**

```python
import math
print(dir(math))  
# 输出：['__doc__', '__loader__', '__name__', '__package__', '__spec__', 'acos', 'acosh', 'asin', ...]

print(dir()) 
# 输出：['__annotations__', '__builtins__', '__doc__', '__file__', '__loader__', '__name__', '__package__', '__spec__', 'math']
```

## 2. 包（Package）

**定义**：包含 `__init__.py` 文件的目录，用于组织多个模块。

### 目录结构

text

my_package/
├── **init**.py # 可以为空，表示这是一个包
├── module_a.py
└── module_b.py

### 导入包中的模块



import my_package.module_a
my_package.module_a.some_function()

# 方式2

from my_package import module_a
module_a.some_function()

# 方式3

from my_package.module_a import some_function
some_function()

### `__init__.py` 的作用

- 标识一个目录是 Python 包

- 可以在其中定义包的初始化代码

- 可以控制 `from package import *` 导入的内容

python

# **init**.py

**all** = ["module_a", "module_b"] # 限制 from my_package import * 的导入范围

## 3. 常用模块导入模式

| 场景         | 推荐写法                        | 原因       |
| ---------- | --------------------------- | -------- |
| 使用模块中的少量函数 | `from math import sqrt, pi` | 代码简洁     |
| 使用模块中的大量函数 | `import math`               | 避免命名冲突   |
| 模块名较长      | `import numpy as np`        | 方便书写     |
| 自定义工具模块    | `import utils`              | 保持命名空间清晰 |

## 4. 避免循环导入

**问题**：两个模块互相导入对方。

**a.py**：

python

import b
def func_a():
 return b.func_b()

**b.py**：

python

import a
def func_b():
 return a.func_a()

**解决方案**：

- 将公共代码提取到第三个模块

- 在函数内部延迟导入

python

# 延迟导入

def func_a():
 from b import func_b
 return func_b()
